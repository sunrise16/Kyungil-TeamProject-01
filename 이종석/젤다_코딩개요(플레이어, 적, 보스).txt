☆ bullet 클래스
* arrow, seed 하위 클래스 작성
  (전에 슈팅게임 예제 하던 것처럼)
* 몬스터가 공격 시 bullet 구조체를 담은 vBullet 벡터 생성
  (발사는 _bullet->fire()로)
  (bullet 클래스에서는 vector<tagBullet>, enemy 클래스에서는 bullet* _bullet, _bullet = new bullet으로 생성)

☆ 링크
* 공격  =  검으로 공격 (상하좌우, 대각선 X)
  (플레이어 RECT 상하좌우에 따로 하나씩 RECT를 별도로 둬서 공격 시 bool 변수 키고 각 방향의 RECT에
   피격 판정 생기게 연출하면 될 거라 생각) 
* 방어  =  방패로 방어 (상하좌우, 대각선 X)
  (특정 키를 누르면 방어 상태가 되게, 이 상태에서 bullet 벡터와 충돌 시 데미지를 입지 않은 상태로 bullet 벡터 제거)
* 이동  =  걷기(프레임렌더 이미지 사용) (상하좌우, 대각선 X)

☆ 일반 적 1 (오크 궁수)
* 활을 쏘는 도중에는 움직이지 않는다
  (이동 모드, 공격 모드에 bool 변수 줘서 체크)
* 화살이 일정거리 이상 가거나 링크에게 피격되면 사라지도록 처리한다
  (일정거리 나아간 후 사라지게 하는 것은 bullet 벡터에 딜레이 변수 추가해서 계속 증가시키게 하면 될듯)

☆ 일반 적 2 (문어)
* 씨앗을 쏜다
  (화살과 같이 bullet 벡터로 생성, 씨앗이 나아가는 방향은 플레이어와 같이 상하좌우 일직선으로)
* 문어의 공격은 방패로 막을 수 있다
  (원작 반영, 방패와 충돌 시 벡터 제거)

☆ 일반 적 3 (빨간 버섯)
* 위로 점프한다
  (점프 표현은 그래픽으로만 처리하고, 실제 코드에서는 bool 변수로 점프 유무 판별)
* 바닥에 닿으면 갈색 먼지구름이 난다
  (점프 bool 변수가 켜졌다가 다시 꺼졌을 때에 한해 주위 몇 칸 이내에 데미지 처리)
* 이동속도가 매우 느리다

☆ 보스
* 처음 등장 시 보스가 작은 상태에서 커진다
  (실제 RECT 크기는 커진 상태에 맞추고, 커지는 것은 그래픽으로만 표현)
* 보스가 손을 양 옆으로 벌리면 눈이 자신의 몸에서 튀어나온다
  (눈의 갯수는 2개)
* 주변에 눈이 빙글빙글 돌면서 보스를 지키는 듯한 형태
  (눈은 별도의 RECT 객체를 가지게 하고, 보스의 주위를 돌게 하는 것은 슈팅게임에서 탄이 휘게 만드는 것을 응용)
  (이건 잘 모르겠으면 팀장에게 문의하셈)
* 눈은 칼로 제거할 수 있다
  (돌고 있는 눈과 공격했을 때 피격 판정이 생긴 RECT가 서로 충돌 시에 눈을 삭제)
* 눈이 검게 변하며 링크를 따라다닌다 눈과 링크가 닿으면 링크가 피격당한다
  (이 부분은 설명이 좀 더 필요할듯.. 보스 주위를 돌고 있던 눈들이 회전을 멈추고 플레이어를 따라오게 하는건지?)
* 검게 변한 눈은 일정 시간(굉장히 짧음) 지나면 깜빡거리며 사라지고 다시 보스의 주변을 떠돈다
  (눈 객체를 삭제한 후 다시 생성하거나 보스의 상하 위치로 다시 돌아오게 하기, 코드 짜는 사람 맘대로)
* 보스는 불을 쏘는데 그 불이 링크에게 닿으면 링크의 옷에 불이 붙는다 (원작)
  (불 붙은 상태 bool 변수로 체크)
* 옷에 불이 붙은 링크는 이동속도가 빨라진다 (다급하게 뛰는 모습) (원작)
  (틱데미지는 fireDelay 같이 딜레이 변수를 줘서 몇초마다 지속딜을 받게 연출)
* 붙은 일정 시간이 지나면 꺼진다 (굉장히 짧음) (원작)
  (위에서 준 fireDelay 값이 일정 수치 이상을 넘어가면 bool 변수 FALSE 줘서 해제)
